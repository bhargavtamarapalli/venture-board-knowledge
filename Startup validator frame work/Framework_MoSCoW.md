# MoSCoW Prioritization Framework

**Version:** 1.0

## Purpose

Prioritize product features and requirements to maximize customer value while controlling MVP scope.

---

## When To Use

- MVP definition
- Product roadmap planning
- Sprint planning
- Feature prioritization
- Stakeholder alignment

---

## Inputs Required

- Product vision
- Customer needs
- Business goals
- Technical constraints

---

## Core Concepts

MoSCoW divides requirements into four categories.

### Must Have
Critical features without which the product fails.

### Should Have
Important features that significantly improve value but are not mandatory for launch.

### Could Have
Useful enhancements that can be deferred if resources are limited.

### Won't Have (This Release)
Features intentionally excluded from the current release.

---

## Prioritization Framework

Evaluate each feature against:

- Customer value
- Problem severity
- Business impact
- Development effort
- Technical risk
- Strategic alignment

---

## Evaluation Checklist

- Does it solve a core customer problem?
- Is it required for MVP?
- Does it increase adoption?
- Can it wait until V2?
- Does it support business goals?

---

## Questions To Ask

- What happens if this feature is removed?
- Will customers refuse to use the product without it?
- Is there a simpler alternative?
- Does it accelerate product-market fit?

---

## Metrics

- Business Impact
- Customer Value
- Development Effort
- Time to Build
- Risk Level
- Adoption Potential

---

## Red Flags

- Everything classified as Must Have
- Feature creep
- Prioritizing opinions over evidence
- Ignoring customer feedback

---

## Best Practices

- Keep MVP as small as possible.
- Prioritize customer outcomes over feature count.
- Reassess priorities after validation.
- Support decisions with evidence.

---

## Common Mistakes

- Building nice-to-have features first.
- Allowing stakeholder bias.
- Expanding MVP scope continuously.
- Ignoring technical dependencies.

---

## Output Format

Produce:

- MoSCoW Prioritization Table
- MVP Feature List
- Deferred Features
- Prioritization Rationale
- Recommended Release Plan
