# Minimum Viable Product (MVP) Framework

**Version:** 1.0

## Purpose

Define the smallest product that delivers meaningful customer value while validating the highest-risk assumptions with minimum time, cost and complexity.

---

## When To Use

- Startup idea validation
- Product planning
- Pre-launch strategy
- Investor due diligence
- Product-Market Fit validation
- Feature prioritization

---

## Inputs Required

- Problem statement
- Ideal Customer Profile
- Customer interviews
- Jobs To Be Done
- Value Proposition
- Market validation results

---

## Core Concepts

An MVP is the minimum solution capable of validating the most critical business assumptions.

The objective is learning, not shipping features.

Every feature should answer one question:

**Does this reduce uncertainty about achieving Product-Market Fit?**

An MVP should:

- Solve one core problem
- Target one customer segment
- Deliver measurable value
- Generate actionable learning
- Be simple to build and iterate

---

## Framework

### Step 1 — Define the Core Problem

Identify the single highest-priority customer problem.

---

### Step 2 — Identify Critical Assumptions

List assumptions related to:

- Customer
- Problem
- Solution
- Pricing
- Acquisition
- Retention

Prioritize assumptions by risk.

---

### Step 3 — Define Core User Journey

Include only the essential workflow required to deliver value.

Remove everything that does not contribute directly.

---

### Step 4 — Prioritize Features

Classify features into:

- Must Have
- Nice to Have
- Future Releases

Use MoSCoW, Kano and RICE to support prioritization.

---

### Step 5 — Define Success Metrics

Measure:

- Activation
- Retention
- Customer Satisfaction
- Conversion
- Learning Objectives

---

### Step 6 — Launch and Iterate

Collect customer feedback.

Validate assumptions.

Iterate rapidly.

---

## Evaluation Checklist

- One customer segment
- One primary problem
- Core workflow defined
- Assumptions documented
- Success metrics established
- Learning objectives clear

---

## Questions To Ask

- What is the smallest product customers would use?
- Which assumption is highest risk?
- Which feature can be removed?
- Does every feature support learning?
- What determines MVP success?

---

## Metrics

- Activation Rate
- Retention Rate
- Time to Value
- Conversion Rate
- Feature Adoption
- Customer Satisfaction
- Interview Success Rate

---

## Red Flags

- Feature-heavy MVP
- Multiple customer segments
- No validation goals
- Long development cycles
- Building before customer evidence

---

## Best Practices

- Optimize for learning, not completeness.
- Ship quickly.
- Validate assumptions continuously.
- Keep development scope intentionally small.

---

## Common Mistakes

- Building Version 1 instead of an MVP.
- Including low-value features.
- Measuring vanity metrics.
- Ignoring customer feedback.

---

## Output Format

Produce:

- MVP Definition
- Core User Journey
- Feature Scope
- Assumption List
- Success Metrics
- Risks
- Recommendations
